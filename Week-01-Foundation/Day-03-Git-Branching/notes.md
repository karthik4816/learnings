 # Branching 
 
Imagine you're writing a book.

You have the original version.

Now you want to add a new chapter, but you don't want to risk breaking the original.

Instead of editing the original directly, you create a copy.

You work on the copy.

If everything looks good, you merge the copy back into the original.

Git works exactly the same way.

A branch is an independent line of development.

# Commands Used in Branching

# 1. To check the current branch
   
   - git branch

 # 2. Creating new branch 

   - git branch <name>

 # 3. Switch branch

   - git switch <name>   

#  4. Merge branch

   - git merge <name>
 
 # 5. Delete branch

   - git branch -d <name>
